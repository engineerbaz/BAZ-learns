# Authentication & Authorization
Manage user access to OpenShift cluster
- Uses a file with user credentials for authentication. 
`htpasswd -c -B -b htpasswd <name> <password>` 
- -c for creating a new file 
- -B decrypt encrption key 
- -b provide password in command line

run `cat htpasswd` to see htpasswd file. 

Run `oc create secret generic <name> --from-file=htpasswd=htpasswd -n openshift-config` to create secrets 

run `oc get oauth cluster -o yaml > oauth.yml`
you can check by `oc explain oauth.spec`

`oc explain oauth.spec.identy` for detailed information about oauth

Edit `oauth.yaml` file and udpate secret location at  `spec.identityProviders.name`

Run `oc replace -f oauth.yml` to update changes.


Alternatively , you may do on Web Console, Login as **Administrator** , go to _Administration_ --> Cluster Settings --> configurtion tab --> Look for  OAuth , Slect htpasswd from list of Identity Provider


## Creating & Deleting Users 

- Regular Users: Standard interactive accounts for devleoper or Administrator
- System USers : Automatically created for infrastructure interaction
- Service accounts: Used by processes running in pods


### Deleting User 
Remove from htpasswd file (delete his line) and update file. 

Get secret from `oc get secret -n openshift-config` after login from kubeadmin as `oc login -u kubeadmin`

Run `oc create secret generic htpasswd-hmcx --from-file=htpasswd --dry-run=client -o yaml -n openshift-config | oc replace -f -`  

Also need to remover identityProviders
Run `oc get user` to get list of user logged in and delete user by `oc delete user <name>`

Check `oc get identity` if user available here, run `oc delete identity htpasswd:<name>`

## Modify User password
- Update htpasswd file `htpasswd -b -B htpasswd <user> <newPassword>`
- 
Run `oc create secret generic htpasswd-hmcx --from-file=htpasswd --dry-run=client -o yaml -n openshift-config | oc replace -f -`
