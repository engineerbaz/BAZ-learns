# Openstack

Mapping between Public Cloud ↔ OpenStack.

## OpenStack Core Services Deep Dive
- Nova (Compute scheduling, hypervisor interaction)
- Neutron (L2/L3, overlays, VXLAN, routing)
- Cinder (block storage architecture)
- Keystone (auth flows)
- Glance (image lifecycle)

| Public Cloud Concept |	OpenStack Equivalent |
| --- | --- |
|EC2 VM |	Nova Instance |
|EBS	| Cinder |
| S3 |	Swift |
| VPC |	Neutron |
| IAM |	Keystone |
| Images (AMI)	| Glance |

---

When a VM is launched, 3 major layers are involved:

- API / Control Layer (Brain)
- Scheduling Layer (Decision maker)
- Data Plane (Actual execution)

In OpenStack: Every service is split into components

`nova-api` → receives request
`nova-scheduler` → decides placement
`nova-compute` → actually runs VM
